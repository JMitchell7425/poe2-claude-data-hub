# PoE2 Build Idea Pack Setup

## Files to add

Add these two files to your GitHub repo:

1. `build_ideas.json`
2. `.github/workflows/build-poe2-idea-packs.yml`

## After adding them

Run:

`Actions → Build PoE2 Idea Packs → Run workflow`

Use defaults:

- `max_per_index`: `120`
- `include_raw_excerpts`: `true`

## Claude Project Knowledge

Keep Claude lean. Add only:

- `build_knowledge/manifest_summary.md`
- `build_knowledge/CLAUDE_DATA_HUB_INSTRUCTIONS.md`
- `session_packs/build_ideas/00-build-idea-registry.md`
- The one pack you are actively using, such as:
  - `session_packs/build_ideas/super-tanky-build.md`
  - `session_packs/build_ideas/caster-build.md`
  - `session_packs/build_ideas/fastest-map-boss-build.md`

Do not add:

- `sources/`
- `build_knowledge/compact/`
- every idea pack at once

## Adding a new build idea

Edit `build_ideas.json`.

Add another object like this:

```json
{
  "slug": "my-new-build",
  "title": "My New Build",
  "purpose": "What this build is supposed to do.",
  "keywords": [
    "keyword one",
    "keyword two",
    "damage type",
    "class",
    "skill",
    "defense",
    "boss",
    "clear"
  ]
}
```

Then rerun:

`Actions → Build PoE2 Idea Packs`
